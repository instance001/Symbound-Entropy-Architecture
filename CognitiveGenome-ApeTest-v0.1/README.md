## 1. The 5 EUCs (v0.1)

Each EUC is produced by a single probe:

1. `RecursionDepth` ‚Äì how far you comfortably track causal chains  
2. `RepresentationPreference` ‚Äì how you naturally explain systems (parts, stories, procedures, etc.)  
3. `CompressionStyle` ‚Äì what survives when you summarise  
4. `AmbiguityTolerance` ‚Äì first move when the brief is fuzzy  
5. `OverloadFailureMode` ‚Äì pattern under too many tasks at once

The detailed prompts and scoring rubrics are defined in the spec document.

````
## 3. Scoring Options

You can score the raw responses either **manually** (human rater) or via an **LLM**.

### 3.1 Manual scoring ‚Äì `ape_score_v0_1.py`

Interactive CLI tool for a human rater.

```bash
cd ApeTest

python ape_score_v0_1.py
````

You‚Äôll be prompted for the path to a raw file, e.g.:

```text
Enter path to raw responses JSON file: cognitive_genome_raw_anon-12ab34cd_20251119T123456Z.json
```

For each probe, the tool:

* Shows the subject‚Äôs text
* Presents rubric-based options
* Asks you to choose:

  * `RecursionDepth` (one)
  * `RepresentationPreference` (1‚Äì2)
  * `CompressionStyle` (1‚Äì2)
  * `AmbiguityTolerance` (one)
  * `OverloadFailureMode` (one)
* Optionally records free-text notes

The output is a **Cognitive Genome v0.1 profile**, e.g.:

```json
{
  "version": "CognitiveGenome_v0.1",
  "subject_id": "anon-12ab34cd",
  "timestamp_utc": "2025-11-19T12:34:56Z",
  "eucs": {
    "RecursionDepth": "Deep",
    "RepresentationPreference": ["SpatialMechanical", "Procedural"],
    "CompressionStyle": ["Structure", "Pattern"],
    "AmbiguityTolerance": "High",
    "OverloadFailureMode": "Scatter"
  },
  "notes": {
    "free_text": "Optional scorer notes here.",
    "rubric": "Scored using Cognitive Genome ‚Äì ApeTest v0.1 manual rubric."
  },
  "source_raw_file": "/absolute/path/to/raw/file.json"
}
```

(Shape taken from the scoring snippet.) 

The profile is written to a file like:

```text
cognitive_genome_profile_<subject_id>_<timestamp>.json
```

### 3.2 LLM-based scoring ‚Äì `ape_score_llm_v0_1.py`

If you want an LLM to apply the rubric automatically:

1. Implement `call_llm(prompt: str) -> str` inside `ape_score_llm_v0_1.py`
   ‚Äì this can call any OpenAI-compatible endpoint or local model.

2. Run:

```bash
cd ApeTest

python ape_score_llm_v0_1.py
```

3. When prompted, provide the raw responses JSON path.

The script:

* Builds a detailed rubric prompt (with definitions of each EUC and label space)
* Sends the prompt + probe responses to your LLM
* Expects a **strict JSON object**:

```json
{
  "RecursionDepth": "Shallow | Medium | Deep | VeryDeep",
  "RepresentationPreference": ["Verbal", "SpatialMechanical", "Narrative", "Procedural", "Metaphorical"],
  "CompressionStyle": ["Structure", "Detail", "Pattern", "Narrative", "EmotionalImpact"],
  "AmbiguityTolerance": "Low | Medium | High",
  "OverloadFailureMode": "Scatter | Freeze | Loop | Oversimplify"
}
```



4. If the JSON parses correctly, a profile file named something like:

```text
cognitive_genome_profile_llm_<subject_id>_<timestamp>.json
```

is written alongside a printed summary of the EUCs.

## 4. Data & File Hygiene

ApeTest produces:

* Raw responses:
  `cognitive_genome_raw_<subject_id>_<timestamp>.json`
* Scored profiles (manual):
  `cognitive_genome_profile_<subject_id>_<timestamp>.json`
* Scored profiles (LLM):
  `cognitive_genome_profile_llm_<subject_id>_<timestamp>.json`

If you‚Äôre using Git, you may want to **ignore** locally generated response/profile files:

```gitignore

cognitive_genome_profile_*.json
cognitive_genome_profile_llm_*.json
```
